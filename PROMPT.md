# תבנית ליצירת שחזור חדש

זה הקובץ שאתה מעתיק לקלוד כשאתה רוצה להוסיף שחזור לארכיון.
**אל תבקש ממנו HTML.** בקש ממנו את קובץ ה-JSON — האתר כבר יודע לרנדר אותו.

---

## מה להדביק לקלוד

> אני מוסיף שחזור לארכיון השחזורים שלי. הנה חומר הגלם:
>
> [כאן: הדבק את השאלות / צלם את הדף / העלה קובץ]
>
> החזר לי **אך ורק קובץ JSON תקין** במבנה הבא, בלי טקסט נוסף ובלי גדרות קוד:
>
> ```json
> {
>   "id": "molecular-2026-moed-a",
>   "subject": "ביולוגיה מולקולרית",
>   "title": "ביולוגיה מולקולרית 2026 — מועד א׳",
>   "kind": "shichzur",
>   "year": 2026,
>   "moed": "א",
>   "added": "2026-08-01",
>   "note": "הערה כללית על השחזור, או null",
>   "questions": [
>     {
>       "q": "טקסט השאלה",
>       "opts": ["מסיח 1", "מסיח 2", "מסיח 3", "מסיח 4"],
>       "a": 2,
>       "note": "הערה לשאלה — קיצורים, או ציון שהשאלה מבוססת גרף. אופציונלי.",
>       "table": {
>         "headers": ["עמודה", "עמודה"],
>         "rows": [["תא", "תא"], ["תא", "תא"]]
>       }
>     }
>   ]
> }
> ```
>
> כללים:
> - `a` הוא **אינדקס מבוסס-אפס** של התשובה הנכונה במערך `opts`. התשובה הראשונה היא 0.
> - `note` ו-`table` אופציונליים — השמט אותם לגמרי אם אין בהם צורך.
> - `kind` הוא אחד מ: `shichzur` (שחזור אמיתי) / `practice` (שאלות תרגול שהמצאת) / `highyield` (נושאים חוזרים).
> - `id` באנגלית בלבד, מקפים במקום רווחים, ייחודי בארכיון.
> - אם שאלה מבוססת על גרף או תמונה שאין לך — כתוב את זה ב-`note` של השאלה.

---

## אחרי שקיבלת את ה-JSON

1. שמור אותו בתיקייה `exams/` בשם `<id>.json` — למשל `exams/molecular-2026-moed-a.json`
2. הרץ בטרמינל, מתוך תיקיית הפרויקט:
   ```
   node sync.js
   ```
   הסקריפט יבדוק שהקובץ תקין (שאין `a` שמצביע לתשובה לא קיימת, שאין מזהה כפול, וכו')
   ויעדכן את `exams/manifest.json` לבד.
3. רענן את האתר. השחזור שם.
