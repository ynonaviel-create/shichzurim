# 🧬 ארכיון השחזורים

אתר אחד שמאחד את כל השחזורים האינטראקטיביים, לכל המקצועות.

## איך זה בנוי

הרעיון המרכזי: **התוכן והמנוע מופרדים.**

עד היום כל שחזור היה קובץ HTML שלם — השאלות היו 5% מהקובץ, וכל השאר CSS ו-JavaScript
שהשתכפלו מחדש בכל פעם. לכן אי אפשר היה לאחד אותם: לא היה מה לאחד, רק ערימת תאומים.

עכשיו:

```
exams/               ← התוכן. כל שחזור = קובץ JSON קטן עם השאלות בלבד.
  manifest.json      ← רשימת המבחנים. נבנה אוטומטית, לא נוגעים בו ידנית.
  biochem-2026-moed-a.json
  ...
assets/app.js        ← המנוע. קורא את כל השחזורים ומרנדר אותם.
assets/style.css     ← העיצוב. משותף לכולם.
index.html           ← המעטפת.
sync.js              ← בודק תקינות ומעדכן את המניפסט.
PROMPT.md            ← התבנית שנותנים לקלוד כדי לייצר שחזור חדש.
```

מכאן, כל שחזור חדש מרוויח אוטומטית כל שיפור שנעשה במנוע — בלי לגעת בשחזורים הישנים.

## הפעלה מקומית

לחיצה כפולה על **`start.command`**. זה פותח את הארכיון בדפדפן.

> אי אפשר פשוט לפתוח את `index.html` בלחיצה כפולה — הדפדפן חוסם קריאת קבצים
> מהדיסק (`file://`) מטעמי אבטחה, והמבחנים לא ייטענו. לכן יש את `start.command`.

## הוספת שחזור חדש

1. פותחים צ'אט עם קלוד ומדביקים את התוכן של `PROMPT.md` יחד עם חומר הגלם.
2. שומרים את ה-JSON שחוזר בתיקייה `exams/` בשם `<id>.json`.
3. מריצים:
   ```bash
   node sync.js
   ```
   הסקריפט בודק שהכול תקין — שאין תשובה נכונה שמצביעה על מסיח שלא קיים,
   שאין מזהה כפול, שיש טקסט לכל שאלה — ומעדכן את המניפסט לבד.
4. מרעננים את האתר. השחזור שם.

## מבנה שאלה

```json
{
  "q": "טקסט השאלה",
  "opts": ["מסיח 1", "מסיח 2", "מסיח 3"],
  "a": 1,
  "note": "אופציונלי — קיצורים, או ציון שהשאלה מבוססת גרף",
  "table": { "headers": ["א", "ב"], "rows": [["1", "2"]] }
}
```

`a` הוא **אינדקס מבוסס-אפס**: התשובה הראשונה היא `0`. זו הטעות הכי קלה לעשות,
ולכן `sync.js` בודק אותה.

## מה יש באתר

- **ספרייה** — כל המבחנים מקובצים לפי מקצוע, עם הציון שלך בכל אחד.
- **תרגול מעורב** — שאלות אקראיות מכמה מבחנים יחד. הדרך הטובה ביותר לבדוק
  אם באמת הפנמת, במקום לזכור את סדר השאלות.
- **הטעויות שלי** — כל השאלות שטעית בהן, מכל המבחנים, במקום אחד.

ההתקדמות נשמרת ב-`localStorage` — כלומר בדפדפן הספציפי שאתה משתמש בו.
היא לא עוברת בין מכשירים, ומי שיכנס לאתר יראה את ההתקדמות של עצמו בלבד, לא שלך.

## העלאה לאוויר (GitHub Pages)

האתר סטטי לגמרי, ולכן אפשר לארח אותו בחינם:

```bash
git init
git add .
git commit -m "ארכיון השחזורים"
gh repo create shichzurim --public --source=. --push
```

ואז ב-GitHub: **Settings → Pages → Source: main branch**.
תוך דקה יש כתובת אמיתית שנפתחת גם בנייד, ושאפשר לשלוח לכל המחזור.

מכאן, הוספת שחזור היא `git push` — וכולם רואים אותו מיד.
