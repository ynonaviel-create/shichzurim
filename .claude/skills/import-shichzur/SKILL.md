# ייבוא שחזור לארכיון

## ⚠️ העיקרון שגובר על הכול

**נאמנות למקור.** אתה **מעתיק**, לא **מייצר**. אם אתה לא בטוח מה כתוב — **תשאל את ינון.
אל תשלים, אל תנחש, אל תשפר ניסוח.** שאלה שהומצאה גרועה משאלה שחסרה.

---

## למה לא לחלץ טקסט מה-PDF

`get_text()` על PDF בעברית **מחזיר את המילים בסדר הפוך**:

> `.המסומן הפחמן תחמוצת דו שחרור ונבדק ... הסובל חולה של שריר ביופסית`

וגם הופך אסימונים לטיניים: `MDM2` → `2.MDM`, `cas9` → `9cas`, `DNMT1` → `1DNMT`.

**זה הרס אמיתי שקרה בפועל** (14/07/2026, ייבוא ביומול). מודל שמקבל טקסט הפוך
**חייב** לשחזר את המשפט מהראש — וזו הדליפה שדרכה נכנסים אי-דיוקים.

**לכן: לא מחלצים טקסט. קוראים את הדף בעיניים.**

## התהליך

### 1. סווג את הקובץ

```bash
python3 -c "
import fitz; d=fitz.open(PATH)
chars=sum(len(d[i].get_text()) for i in range(d.page_count))
imgs=sum(len(d[i].get_images(full=True)) for i in range(d.page_count))
print(f'{d.page_count} עמ׳ | {chars} תווים | {imgs} תמונות')
"
```
לא משנה אם טקסט או סריקה — **בשני המקרים קוראים ויזואלית.**

### 2. רנדר לתמונות וקרא

```bash
python3 -c "
import fitz; d=fitz.open(PATH)
for i in range(d.page_count):
    d[i].get_pixmap(dpi=140).save(f'{SCRATCH}/p{i+1}.png')
"
```
ואז `Read` על כל עמוד. **קרא את כולם — כולל האחרון.**

### 3. חפש מפתח פתרונות

שחזורים רבים כוללים **טבלת תשובות בעמוד האחרון**. זו אמת מוחלטת — **השתמש בה.**
אל תסיק תשובות מהידע שלך אם יש מפתח.

סימונים שכיחים במפתח:
| סימון | משמעות | מה לעשות |
|---|---|---|
| `א?` | גם המשחזר לא היה בטוח | השאר, **הוסף `note`** שמזהיר |
| `-` | אין תשובה | **השמט את השאלה** |

### 4. סמן שאלות פגומות

| בעיה | פעולה |
|---|---|
| פחות משני מסיחים | **השמט** + תעד ב-`note` של המבחן |
| כתוב "לא שוחזר כראוי" | **השמט** |
| שני מסיחים זהים | **השאר כמו שהוא** + `note` על השאלה |
| שאלה נשענת על גרף | חלץ תמונה (סעיף 6) |

### 5. חלץ תמונות (גרפים)

```bash
python3 -c "
import fitz; d=fitz.open(PATH); p=d[PAGE]
for i,inf in enumerate(p.get_images(full=True)):
    img=d.extract_image(inf[0])
    open(f'assets/img/{EXAM_ID}-q{N}.{img[\"ext\"]}','wb').write(img['image'])
"
```
ואז בשאלה: `"image": "assets/img/<exam-id>-q<n>.jpeg"`

**אם השאלה בלתי פתירה בלי הגרף — חובה לחלץ.** אל תסתפק בהערה.

### 6. מזג הסברים מג'מיני (אם יש)

לג'מיני יש חיבור ל-NotebookLM עם חומרי הקורס — **יתרון אמיתי** על פני קלוד
לצורך `explain` ו-`topic`. **אבל אין לסמוך עליו לטקסט או לתשובות.**

| שדה | מקור |
|---|---|
| `q`, `opts` | **הקריאה הויזואלית שלך** מה-PDF |
| `a` | **מפתח הפתרונות** מהשחזור |
| `topic`, `explain` | ג'מיני |

**⚠️ ג'מיני סופר מ-1, לא מ-0.** תמיד המר, ותמיד **הצלב מול המפתח**.

### 7. דו"ח הטיה (חובה בשחזורי סטודנטים)

שחזור לא-רשמי נוטה להטיה: מי ששחזר זכר את **התשובה הנכונה** במלואה, ואת
**המסיחים** בקיצור. **מדוד ורשום בהערת המבחן:**

```bash
node -e "
const qs=require('PATH').questions;
let long=0; qs.forEach(q=>{const l=q.opts.map(o=>o.length),m=Math.max(...l);
  if(l[q.a]===m && l.filter(x=>x===m).length===1) long++;});
console.log('הנכונה היא הארוכה ביותר: '+Math.round(long/qs.length*100)+'% (מקרי ≈25%)');
"
```
**אל תתקן את השאלות.** תיקון = כתיבה מחדש = זה כבר לא שחזור. **רק תעד.**

### 8. הכרע את החזרות (High Yield)

`sync.js` מריץ את `repeats.js`, שמזהה שאלות שחוזרות בין מחזורים ובונה מהן את
מבחן ה-High Yield. **שחזור חדש יוצר זוגות חדשים שהמכונה לא יכולה להכריע לבד:**

```bash
node repeats.js --review
```

שני סוגי הכרעות, ושתיהן נרשמות ל-`exams/repeats-ledger.json` ונשמרות **לתמיד**:

| שאלה | תשובות | לאן |
|---|---|---|
| זו אותה שאלה? | `true` / `false` | `pairs` |
| למה המפתחות שונים? | `true` (אותה תשובה, ניסוח אחר) · `"opts"` (המסיחים היו שונים) · `false` (סתירה אמיתית) | `answers` |

**כלל ההכרעה של ינון:** *"אם העיקרון דומה — זו אותה שאלה, גם אם מילה פה ומילה
שם שונות. אלא אם כן המילה הזאת הופכת את כל העיקרון."* למשל **`false`**: "השלב
הראשון ב-PCR" מול "השלב **השני**"; מודל **Billboard** מול **Enhanceosome**.

**עד שהוכרע — לא מוצג כלום.** תג "חזרה 3 פעמים" שהוא שקר גרוע מהיעדר תג.

**⚠️ אל תיתן למכונה להכריע בסתירות.** נוסה בפועל וזה נכשל: השוואת מחרוזות ראתה
ב"מתחלק בתדירות **גבוהה**" ו"מתחלק בתדירות **נמוכה**" את *אותה* תשובה. גם
היפוך ("מה נכון" מול "מה **אינו** נכון") וגם "כל התשובות נכונות" מייצרים
סתירות מדומות.

### 9. מי צודק? — שאלה לנוטבוק, לא לך

כשהמפתחות באמת סותרים, **"איזו תשובה נכונה" היא שאלת תוכן.** אל תכריע אותה
מהידע שלך ואל תבקש מינון להכריע מהזיכרון — **ל-NotebookLM שלו מחוברים חומרי
הקורס**, והוא הפוסק.

```bash
node repeats.js --ask     # בונה פרומפט עצמאי ושומר ב-sources/ask-notebooklm.md
```

ינון מדביק בנוטבוק, וההכרעה נרשמת ב-`repeats-ledger.json` תחת `truth`:

```json
"truth": {
  "<מזהה הזוג>": { "answer": "<נוסח המסיח הנכון>", "why": "<הנימוק>" }
}
```

אז המבחן **מתקן את המפתח בפועל** — לא רק מעיר. אם התשובה שהוכרעה הוצעה רק
באחד המחזורים, הנציג במבחן מוחלף לאותו מחזור.

**זה הכלל הכללי:** כל התלבטות שדורשת ידע בחומר → **נסח כפרומפט מוכן לנוטבוק.**
התלבטות טקסטואלית ("זו אותה שאלה?") → אתה מכריע.

## סכימת השאלה

```json
{
  "q": "טקסט השאלה",
  "opts": ["מסיח 1", "מסיח 2", "מסיח 3", "מסיח 4"],
  "a": 2,
  "topic": "שם נושא — מרשימה סגורה, עקבי בין מבחנים",
  "explain": "הסבר קצר",
  "source": "שחזור מחזור נ״א",
  "note": "הערה על בעיה במקור",
  "image": "assets/img/exam-q7.jpeg",
  "table": { "headers": ["א","ב"], "rows": [["1","2"]] }
}
```

`a` = **אינדקס מבוסס-אפס**.

`qid` ו-`repeat` נכתבים אוטומטית ע״י `repeats.js` — **אל תיגע בהם ידנית.**

## שדה `trust` — חובה בכל שחזור

כמה המפתח אמין, לפי הצהרת המשחזרים עצמם. **מכריע כשמחזורים חלוקים על תשובה.**

| ערך | מתי |
|---|---|
| `verified` | "כל התשובות אומתו בחשיפה" |
| `partial` | מפתח מהשחזור, או "אומת" אך נמצאו בו טעויות |
| `unverified` | "התשובות לא אומתו" / רובן משוערות |
